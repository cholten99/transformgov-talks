TRANSFORMGOV TALKS — POST-EVENT UPDATE GUIDE
============================================

Do these steps after each event. Steps 1-4 update the dashboard page.
Step 5 adds the event to the Previous Events page.

All of this is done by giving Claude the relevant file or instruction.
Claude will SSH into the server and run the scripts.


STEP 1: REGISTRATIONS (Luma)
-----------------------------
Get the guest list from Lu.ma:
  Lu.ma → [your event] → Guests → Export CSV

Give the CSV to Claude and say:
  "Please add this Luma event. Event name: [e.g. June 2026], date: [e.g. 2026-06-19]"

Claude will copy the CSV to the server and run:
  python3 sync_luma.py --file <csv> --event-name "June 2026" --event-date 2026-06-19

Then delete the CSV — it contains email addresses.


STEP 2: PODCAST DOWNLOADS (Libsyn)
------------------------------------
Export from Libsyn:
  Libsyn → Reports → Episode Downloads → Export (CSV or ZIP)

Give the file to Claude and say:
  "Please update the podcast data from [filename]"

Claude will copy the file to the server and run:
  python3 sync_libsyn.py --file <filename>

This replaces all podcast data each time (Libsyn totals update over time).
Then delete the file.


STEP 3: MAILING LIST (Kit)
---------------------------
No file needed. Tell Claude:
  "Please sync the Kit mailing list"

Claude will run:
  python3 sync_kit.py


STEP 4: YOUTUBE
---------------
No file needed. Tell Claude:
  "Please sync YouTube"

Claude will run:
  python3 sync_youtube.py


STEP 5: PREVIOUS EVENTS PAGE
-----------------------------
The previous events page is generated from previous_events.yml.
Add the new event as a new entry at the TOP of the events list in that file.

Each entry follows this format:

  - month: June 2026
    items:
    - Speaker One Name, Role, Organisation, on Talk Title
    - Speaker Two Name, Role, Organisation, on Talk Title
    links:
      blog: https://medium.com/@transformgovtalks/june-2026-write-up-...
      youtube: https://youtu.be/...
      spotify: https://open.spotify.com/episode/...
      apple podcasts: https://podcasts.apple.com/...

Notes:
- All four link types are optional — omit any that don't exist yet
- Items are bullet points describing each speaker/talk
- Month should be written as "Month YYYY" (e.g. "June 2026")

Once the YAML is updated, tell Claude:
  "Please rebuild the previous events page"

Claude will run:
  python3 build_previous.py \
    --input-yaml previous_events.yml \
    --template previous_template.shtml.j2 \
    --output previous.shtml


HOW THE DASHBOARD WORKS
------------------------
There is no database. Each sync script fetches or reads source data and
writes a JSON file that the dashboard reads directly in the browser:

  data/subscribers.json  ← Kit mailing list      (sync_kit.py)
  data/videos.json       ← YouTube stats         (sync_youtube.py)
  data/libsyn.json       ← Podcast downloads     (sync_libsyn.py)
  data/luma.json         ← Event registrations   (sync_luma.py)

The private/ directory holds all past Luma guest emails for tracking
unique attendees across events. It is blocked from web access.

Kit and YouTube can be synced at any time, not just after events.
